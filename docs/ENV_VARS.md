# Переменные окружения (.env)

Этот файл описывает все ключевые переменные окружения, которые используются в проекте, и какие значения они ожидают.

Для актуального списка смотрите `.env.example` — он синхронизирован с этим описанием.

## 1. Базовые настройки проекта

- **`PROJECT_NAME`**
  - **Назначение:** Имя проекта (используется в заголовке FastAPI и логах).
  - **Тип:** строка.
  - **Пример:** `Invext`.

---

## 2. База данных

- **`DATABASE_URL`**
  - **Назначение:** Строка подключения к PostgreSQL, которую используют backend и Alembic.
  - **Тип:** URL для SQLAlchemy async (driver `asyncpg`).
  - **Формат:**  
    `postgresql+asyncpg://USER:PASSWORD@HOST:PORT/DB_NAME`
  - **Примеры:**
    - В Docker‑сети: `postgresql+asyncpg://user:password@db:5432/invext`
    - Локально (если Postgres на хосте): `postgresql+asyncpg://user:password@localhost:5432/invext`

---

## 3. Backend / приложение

- **`BACKEND_HOST`**
  - **Назначение:** Адрес, на котором FastAPI слушает внутри контейнера / при локальном запуске.
  - **Тип:** строка (IP или `0.0.0.0`).
  - **Ожидаемое значение:** обычно `0.0.0.0`.

- **`BACKEND_PORT`**
  - **Назначение:** Порт, на котором запущен FastAPI внутри контейнера / локально.
  - **Тип:** число.
  - **Ожидаемое значение:** `8000` (по умолчанию, но может быть изменён).

- **`ADMIN_API_KEY`**
  - **Назначение:** Секретный ключ для legacy‑админ‑эндпоинтов (`/v1/admin/*`), которые дергает бот через заголовок `X-ADMIN-KEY` или `Authorization: Bearer`.
  - **Тип:** строка (секрет).
  - **Ожидаемое значение:** длинная случайная строка, не публикуется в репозитории.

---

## 4. Админ‑ключи и роли

- **`ADMIN_TELEGRAM_IDS`**
  - **Назначение:** Список Telegram ID админов, которые имеют доступ к админ‑функциям в боте и/или админ‑панели.
  - **Тип:** строка, список чисел через запятую.
  - **Пример:** `ADMIN_TELEGRAM_IDS=123456789,987654321`

- **`ADMIN_JWT_SECRET`**
  - **Назначение:** Секрет для подписи JWT‑сессий HTML‑админ‑dashboard (`/database`). Используется только на backend.
  - **Тип:** строка (секрет).
  - **Ожидаемое значение:** длинная случайная строка (минимум 32 символа), не публикуется.
  - **Пример:** `ADMIN_JWT_SECRET=3a0f2f3f7c2a4e...`

- **`ADMIN_ALLOWED_IPS`**
  - **Назначение:** (Опционально) ограничение по IP для доступа к `/database/api/*`.
  - **Тип:** строка, список IP‑адресов через запятую.
  - **Ожидаемое значение:**
    - Пусто (`""`) — доступ с любых IP.
    - Или, например: `ADMIN_ALLOWED_IPS=1.2.3.4,5.6.7.8`

## 5. Telegram‑бот

- **`BOT_TOKEN`**
  - **Назначение:** Токен Telegram‑бота, выданный `@BotFather`.
  - **Тип:** строка (секрет).
  - **Формат:** `1234567890:AA...`.
  - **Ожидаемое значение:** действительный токен вашего бота, который не должен попадать в публичный репозиторий.
  - **Дополнительно:** используется не только самим ботом, но и backend‑сервисом сделок для отправки Telegram‑уведомлений о новой сделке, закрытии сбора и начисленной прибыли.

- **`BACKEND_BASE_URL`**
  - **Назначение:** Базовый URL backend, по которому бот обращается к API.
  - **Тип:** URL.
  - **Примеры:**
    - В Docker‑сети: `http://app:8000`
    - При локальной отладке бота: `http://localhost:8000`

## 6. Лимиты по операциям

Эти переменные задают минимальные и максимальные суммы для проверок в сервисах депозитов/выводов.

- **`MIN_DEPOSIT`**
  - **Назначение:** Минимально допустимая сумма пополнения (USDT).
  - **Тип:** число (может быть целым или с плавающей точкой, но обычно целое).
  - **Пример:** `1`

- **`MAX_DEPOSIT`**
  - **Назначение:** Максимально допустимая сумма пополнения (USDT).
  - **Тип:** число.
  - **Пример:** `100000`

- **`MIN_WITHDRAW`**
  - **Назначение:** Минимально допустимая сумма вывода (USDT).
  - **Тип:** число.
  - **Пример:** `1`

- **`MAX_WITHDRAW`**
  - **Назначение:** Максимально допустимая сумма вывода (USDT).
  - **Тип:** число.
  - **Пример:** `100000`

## 7. Crypto Pay

- **`CRYPTO_PAY_TOKEN`**
  - **Назначение:** API‑токен приложения в Crypto Pay API (`@CryptoBot` / `@CryptoTestnetBot` → Crypto Pay → My Apps).
  - **Тип:** строка (секрет).
  - **Ожидаемое значение:** токен вида `123456:AA...`, который вы видите в настройках приложения Crypto Pay.

- **`APP_URL`**
  - **Назначение:** Публичный HTTPS‑URL backend, по которому Crypto Pay может вызывать webhook `/crypto/webhook`, а также базовый URL, который используется в боте для формирования ссылки на админ‑dashboard (`APP_URL/database`).
  - **Тип:** URL.
  - **Примеры:**
    - Прод: `https://your-domain.com`
    - Тест на ngrok: `https://abcd1234.ngrok.app`
    - Локально для разработки: `http://localhost:8000` (в этом случае ссылка на админку в боте будет `http://localhost:8000/database`).
  - **Важно:** этот URL должен быть доступен из интернета и указывать на фронтовой reverse‑proxy (например, Nginx), который проксирует `/crypto/webhook` на backend.

1. Добавьте их в `.env.example` с комментариями.
2. Обновите это описание (`docs/ENV_VARS.md`), чтобы было понятно назначение и ожидаемый формат значений.
